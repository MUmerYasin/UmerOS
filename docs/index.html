Quick Start
Get Umer OS running on your computer in 5 minutes.
⚠ Prerequisites
Python 3.10 or higher is required. Install from python.org. For post-quantum crypto: pip install cryptography. For quantum simulation: pip install numpy.
1

Download and Extract

Download UmerOS.zip and extract it to a folder of your choice.

unzip UmerOS.zip
cd UmerOS
2

Install Dependencies

Install the two required Python packages (everything else uses the standard library).

pip install numpy cryptography
3

Run the Test Suite

Verify your installation — all 305 tests should pass.

python -m unittest discover -s tests -v
4

Simulate the Boot Sequence

Watch Umer OS boot — system check, kernel verification, and kernel initialisation.

python -m boot.bootloader
5

Launch the Desktop (Optional)

Install Kivy for the full graphical shell. Without Kivy, headless mode is used.

pip install kivy
python -m ui.gui

Quantum Computing in Umer OS
How quantum-inspired features work today — and what's planned for the future.
💡 What "Quantum-Inspired" Means
True quantum computers with physical qubits are not yet available in consumer devices. Umer OS simulates quantum behaviour on classical hardware using NumPy mathematics. This gives us quantum-inspired algorithms today, while the API is ready for real QPU hardware tomorrow.
⚛

Superposition-Inspired Scheduling

Each task in the scheduler has a "superposition" probability score (0.0–1.0) computed by the AI manager. The quantum simulator can refine these scores using Hadamard gate sampling, ensuring better CPU allocation than simple round-robin.

# QuantumCircuitSimulator usage
sim = QuantumCircuitSimulator(n_qubits=2)
sim.apply_h(0)          # superposition
sim.apply_cnot(0, 1)    # entanglement
bit = sim.measure()     # collapse to 0 or 1

Post-Quantum Cryptography

Umer OS uses CRYSTALS-Kyber for key exchange and CRYSTALS-Dilithium for signatures — the NIST-standardised algorithms that are secure even against future quantum computers. Falls back to Ed25519/AES-256-GCM if liboqs is not installed.

pqc = PostQuantumCrypto()
pk, sk = pqc.generate_keypair()
ct = pqc.encrypt(b"secret", pk)
pt = pqc.decrypt(ct, sk)   # = b"secret"
sig = pqc.sign(b"msg", sk)
ok  = pqc.verify(b"msg", sig, pk)

AIResourceManager

Uses Exponentially-Weighted Moving Average (EWMA) to predict a process's CPU usage, RAM usage, and scheduling success probability. Feeds directly into the HybridScheduler to prioritise tasks that are likely to succeed.

arm = AIResourceManager(window=20)
arm.record_cpu(pid=1, usage=0.85)
pred = arm.predict_cpu_usage(pid=1)
# pred ≈ 0.85 (converges to recorded value)

AIFirewall

Analyses each process's system call patterns. Unusual combinations (ptrace + setuid + execve) score high on the anomaly scale (0.0 = safe, 1.0 = critical). Processes above the threshold (default 0.75) are automatically quarantined.

fw = AIFirewall(threshold=0.75)
fw.profile_process(pid=3)
score = fw.check_and_act(pid=3,
    syscall_trace=["read","write","stat"])
# score ≈ 0.0 → safe

Security Architecture
Zero-trust by default. Every process, every message, every boot image is verified.
🔑

Capability-Based Access

No process has any permission by default. Every capability (fs.read, net.send, gpu.render…) must be explicitly granted. Revocation is instant. check() raises PermissionError immediately if denied.

✍

HMAC-Signed IPC

Every inter-process message is signed with HMAC-SHA256 using a per-bus random 32-byte key. Unsigned messages are silently dropped. The bus also provides a sign(payload) utility for external callers.

🔏

Post-Quantum TLS

All network connections use CRYSTALS-Kyber for key exchange and Dilithium for signatures — algorithms that remain secure even against a quantum computer running Shor's algorithm.

API Reference
Key classes and methods for developers building on Umer OS.

UmerKernel

from kernel.umer_kernel import UmerKernel
import asyncio

kernel = UmerKernel(total_memory_bytes=512*1024*1024)

async def main():
    await kernel.init()

    # Spawn a process
    pid = kernel.spawn_process(
        name="my_service",
        priority=0.8,
        capabilities=["fs.read","net.send"]
    )

    # Check system health
    print(kernel.status())
    print(f"Uptime: {kernel.uptime():.2f}s")

    # Kill a process
    kernel.kill_process(pid)

    await kernel.shutdown()

asyncio.run(main())

QFS Filesystem

from fs.qfs import QFS

qfs = QFS(max_store_bytes=256*1024*1024)
qfs.mount("/")

# Write a file
addr = qfs.write_file(
    "/home/user/report.txt",
    b"Quarterly report data..."
)

# Read it back
data = qfs.read_file("/home/user/report.txt")

# Semantic search
results = qfs.search("quarterly report")

# Take a snapshot (for rollback)
snap = qfs.snapshot()
qfs.write_file("/home/user/report.txt", b"Modified")
qfs.restore_snapshot(snap)  # undo changes

# Check storage savings
stats = qfs.stats()
print(f"Saving: {stats['space_saving_pct']}%")

# Create Bell state (entanglement)

sim.apply_cnot(0, 1) # Entangle qubit 0 and 1

# Measure

# → [0.5, 0.0, 0.0, 0.5]

Post-Quantum Crypto

from quantum.crypto_pqc import PostQuantumCrypto

pqc = PostQuantumCrypto()
print(f"Backend: {pqc.backend}")
# → "liboqs" or "fallback"

# Key pair generation
pk, sk = pqc.generate_keypair()

# Encrypt / Decrypt
ciphertext = pqc.encrypt(b"secret data", pk)
plaintext  = pqc.decrypt(ciphertext, sk)
assert plaintext == b"secret data"

# Sign / Verify
signature = pqc.sign(b"kernel image", sk)
is_valid  = pqc.verify(b"kernel image",
                        signature, pk)
assert is_valid  # True
Frequently Asked Questions
Answers for both technical and non-technical users.

❓ Is Umer OS a real operating system I can install?

Yes and no. The core software works today on any computer with Python 3.10+. The full OS install (replacing your existing OS) is experimental — recommended for developers and enthusiasts only. Most users should run it as a Python environment first.

❓ Does it really use quantum computing?

Today, all quantum features are simulated using NumPy mathematics on classical hardware. This gives us quantum-inspired algorithms (better scheduling, stronger crypto) that work right now. Real QPU hardware integration is planned for the future.

❓ Can I run Windows apps (.exe)?

Yes — if Wine is installed on your system. The WineShim in the compatibility layer sets up a per-app wineprefix and launches Windows executables. This is marked EXPERIMENTAL as not all apps work perfectly.

❓ Can I run Android apps (.apk)?

Yes — if ADB is installed and a device/emulator is connected. The AndroidContainer uses adb shell am start to launch APKs. This is EXPERIMENTAL and requires an Android runtime environment.

❓ What happens to my data?

Nothing without your permission. All AI features are opt-in. No data is ever sent anywhere without explicit consent from the AIGovernance module. You can call AIGovernance.clear_all() at any time to erase all learned preferences.

❓ Can I install this on my iPhone?

No. Apple's Secure Enclave hardware chip and Verified Boot chain make it physically impossible to run a third-party OS without jailbreaking, which voids your warranty and is against Apple's terms of service. Focus is on Android and desktop platforms.

❓ How do I contribute to Umer OS?

All code is in Python and well-documented. Start by running the test suite, then pick a subsystem (quantum, AI, security, etc.) and look for TODO comments. Submit pull requests with tests — all PRs must maintain the 305-test pass rate.

❓ What hardware does it support?

Umer OS runs on x86_64 (Intel/AMD), ARM64 (Apple Silicon, Raspberry Pi, Android phones), and in QEMU virtual machines. RISC-V support is planned for the future. Minimum: 512 MB RAM, Python 3.10+.
